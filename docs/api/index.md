# 런치랩 API 문서

## 개요

이 문서는 런치랩 서비스의 API에 대한 종합적인 가이드를 제공합니다. 런치랩은 급식 및 식자재 관리를 위한 마켓플레이스 모듈 시스템을 제공하며, 이 문서는 해당 API를 사용하는 방법을 설명합니다.

## API 문서 목차

### 마켓플레이스 모듈 API

- [마켓플레이스 모듈 API](./marketplace-modules.md) - 마켓플레이스 모듈 관리 API
  - 모듈 목록 조회
  - 특정 모듈 조회
  - 모듈 구독 정보 조회
  - 모듈 설정 정보 조회 및 업데이트

### 회사별 모듈 관리 API

- [회사별 모듈 API](./company-modules.md) - 회사별 모듈 구독 및 관리 API
  - 회사 모듈 목록 조회
  - 회사에 모듈 구독 추가
  - 회사 특정 모듈 구독 정보 조회
  - 회사 모듈 구독 취소
  - 회사 모듈 설정 조회 및 업데이트

## 인증

모든 API 요청은 Clerk을 통한 인증이 필요합니다. API 호출 시 인증된 사용자의 JWT 토큰이 요구됩니다.

## 에러 처리

API는 다음과 같은 형식의 에러 응답을 반환합니다:

```json
{
  "error": "에러 메시지"
}
```

### 일반적인 에러 코드

- **400 Bad Request**: 잘못된 요청 형식 또는 필수 파라미터 누락
- **401 Unauthorized**: 인증되지 않은 요청
- **403 Forbidden**: 접근 권한 없음
- **404 Not Found**: 리소스를 찾을 수 없음
- **500 Internal Server Error**: 서버 내부 오류

## API 버전 관리

현재 API 버전은 v1입니다. 모든 API 요청은 최신 버전으로 라우팅됩니다.

## 데이터 형식

모든 요청 및 응답은 JSON 형식입니다. 요청 본문의 `Content-Type`은 `application/json`이어야 합니다.
